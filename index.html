Description of the Existing System

The current Teaching and Learning Support System provides essential functionality for resource sharing between educators and students. It allows instructors to upload materials, enabling students to access notes, presentations, and assignments, which serves as a basic repository for course content. However, this structure focuses heavily on content storage and retrieval, offering limited interaction and analytics. Understanding its core functionality, strengths, weaknesses, and security is essential to identify gaps that the new system can address.

2.1.1 Major Functions of the Existing System

The existing system primarily functions as a central content repository where users can input and retrieve educational materials.

  1. Input Process: User input revolves around account creation and login, resource uploading by instructors, and resource access by students. Educators upload documents, videos, and other learning aids, while students use the system to access these materials. Additionally, students can provide minimal feedback, often in the form of ratings or simple comments on the content. However, there is no option for students or teachers to interact directly through the platform, which limits collaborative learning and direct feedback mechanisms.

  2. Processing Mechanism: Once data is uploaded, the system processes it primarily for storage and access, with minimal categorization or indexing for efficient retrieval. This process involves basic user authentication and user-data management. The system does offer a rudimentary form of tracking, recording which resources users have accessed, but lacks real-time analytics or tracking for individual progress.

  3. Output: The output primarily includes access logs showing which materials were used and by whom, giving administrators a view of system usage. The system also provides limited reporting, mainly focused on login frequency and access records. However, it lacks advanced reporting or insights into student engagement with specific topics, making it difficult for educators to tailor content or track student progress accurately.

2.1.2 Weaknesses and Strengths of the Existing System

The existing system does have strengths, but it also faces several limitations that impact its functionality and user satisfaction.

  1. Strengths:

    One of the main advantages of the current system is its simplicity and ease of use, which allows users to navigate without extensive training. The platformâ€™s user-friendly design makes it easy for teachers to upload and organize materials and for students to access these resources. This straightforward design fulfills the minimum requirements for sharing educational content. Additionally, by centralizing resources, it creates a unified place where educational materials are stored, reducing the need for external tools for resource management.

  2. Weaknesses:

    Despite its simplicity, the system lacks several critical functions. The limited interaction options mean that students and teachers are unable to engage directly within the platform. This absence of interactivity diminishes the learning experience and restricts collaborative opportunities. The system's tracking capabilities are also insufficient, as it cannot monitor detailed user behavior, individual learning progress, or performance analytics. Without these insights, educators find it difficult to identify and address student needs. Moreover, the limited categorization system hinders efficient retrieval of resources, often making it challenging for users to locate specific materials without a highly efficient search mechanism.

2.2.3 Performance of the Existing System

While the system performs reliably under light loads, it struggles with scalability as the number of users and resources grows. High activity levels, such as during peak times at the beginning or 17 end of terms, can slow down response times and lead to delays in accessing content. Additionally, the lack of an optimized search function and insufficient indexing further contribute to performance issues, particularly as the volume of stored resources increases. Users may experience slow retrieval times, which negatively impacts the usability of the platform.

2.2.4 Security and Control

Security within the current system is basic, relying primarily on password-protected user accounts for access control. Neither user data is not strongly encrypted, nor are there measures such as two-factor authentication nor advanced data security protocols in place, leaving sensitive information vulnerable to unauthorized access. The absence of role-based access control (RBAC) means that users have uniform access rights, which restricts the system's flexibility in managing different user groups (e.g., administrators vs. students). Furthermore, without strong audit trails, the system lacks the ability to monitor and log detailed user activity, making it difficult to detect suspicious behavior or address potential security breaches proactively.

2.2.2 Requirements of the Proposed System

The proposed Teaching and Learning Support System is designed to enhance the educational experience by meeting a comprehensive set of functional and non-functional requirements. These requirements ensure that the system will provide robust and reliable support to students, faculty, and administrators, ultimately promoting a more effective and efficient educational environment.

2.2.2.1 Functional Requirements

The functional requirements outline the key capabilities that the system must provide to fulfill its intended purpose within an educational setting. These include:

  • User Authentication and Role-Based Access: The system should offer secure login and access control mechanisms, allowing students, faculty, and administrators to access different features based on their roles. Students need access to course materials, assignment submission, and communication tools, while faculty members require access to tools for managing courses, grading, and providing feedback. Administrators, on the other hand, will have access to system settings and data reporting functionalities.

  • Course Content Management: Faculty members must have a streamlined interface to upload, edit, and organize course materials, including lecture notes, readings, multimedia resources, and assignment instructions. This ensures that students can conveniently access all necessary resources, enhancing their learning experience and allowing faculty to keep content up-to-date easily.

  • Assessment, Grading, and Feedback: The system should include tools for online assessments, including quizzes and tests, that allow for automated grading when applicable. Faculty members should be able to provide both quantitative (grades) and qualitative (feedback) assessments, ensuring that students receive comprehensive insights into their academic progress and areas for improvement.

  • Communication and Notifications: Real-time communication is essential in a learning environment. The system should support messaging between students and faculty, group discussions, and notifications for important updates, deadlines, and announcements. This promotes consistent engagement and timely information sharing, reducing potential miscommunication.

  • Data Collection and Reporting: The system should gather data on student engagement, attendance, and performance, enabling faculty and administrators to generate reports and track academic progress. This data can be analyzed to identify trends, such as areas where students may be struggling, and to make informed decisions to enhance educational outcomes.

2.2.2.2 Non-Functional Requirements

  1. Performance: The system should deliver a fast response time, allowing students and faculty to access materials, submit assignments, and view feedback with minimal delay. It should support multiple concurrent users and ensure smooth performance, even during peak usage periods.

  2. Scalability: The system must handle increasing data volumes and user growth, adapting to the needs of expanding educational institutions and a larger student population without compromising performance.

  3. Usability: Designed for both tech-savvy and non-technical users, the system should have an intuitive, user-friendly interface. This includes easy navigation and clear instructions to enhance accessibility and minimize user error.

  4. Reliability: To support continuous learning, the system should be operational and available 24/7. Downtime should be minimized, and scheduled maintenance should occur during off-peak hours.

  5. Maintainability: The system should be straightforward to maintain, with comprehensive documentation for administrators and support staff. Updates and improvements should be easily implemented without disrupting service.

  6. Compatibility: The system must work across various devices, platforms, and operating systems, including desktops, tablets, and mobile devices, to accommodate diverse user preferences and access points.

  7. Security: Data protection is critical, requiring robust encryption, role-based access control, and multi-factor authentication to prevent unauthorized access and safeguard sensitive student and institutional information.

  8. Backup and Recovery: Regular data backups are essential, ensuring that data can be restored quickly in the event of system failures or data loss. The system should have a well-defined disaster recovery plan to protect data integrity and minimize downtime.

  9. Availability: The system should ensure a high availability rate, preferably 99.5%, to support continuous access to learning materials and resources for students and faculty.

  10. Resource Efficiency: Efficient resource usage, including CPU, memory, and storage optimization, is essential to ensure stable performance while managing the system's infrastructure costs effectively.
